## Background

Two months ago, while planning a trip, I realized that existing travel planning tools were either too rigid or too generic. I wondered: what if an AI could understand my preferences, budget, and time constraints, and generate a personalized itinerary?

More importantly: **could I use AI itself to build this AI tool?**

This became my first Vibe Coding experiment.

## What Is Vibe Coding?

If you're not familiar with the term, Vibe Coding means:

> Instead of writing code directly, you describe what you want (prompt engineering) and let AI generate the code. Your role shifts from "typist" to "director" — you articulate the vision, and AI handles the implementation details.

The core workflow:

1. **Describe the goal** — tell AI what you want in natural language
2. **Review the output** — inspect the generated code
3. **Provide feedback** — point out issues, request improvements
4. **Iterate** — repeat until satisfied

## Tech Stack (Suggested by Cursor)

When I described my idea to Cursor, it recommended this stack:

```typescript
const techStack = {
  framework: "Next.js 14 + App Router",
  styling: "Tailwind CSS + shadcn/ui",
  ai: "OpenAI API (GPT-4o)",
  database: "Supabase (Postgres)",
  auth: "NextAuth.js",
  deployment: "Vercel",
};
```

I barely modified this suggestion — it was exactly what I would have picked myself.

## The Process

### Day 1 (Saturday)

- **Morning (2h)**: Generated project scaffold with Cursor, including routing and layouts
- **Afternoon (3h)**: Implemented AI chat interface, connected to OpenAI API
- **Evening (2h)**: Built itinerary display component with drag-and-drop

### Day 2 (Sunday)

- **Morning (2h)**: Added Supabase for data persistence
- **Afternoon (2h)**: Polished UI with animations and responsive design
- **Evening (1h)**: Deployed to Vercel with custom domain

**Total: ~12 hours**, with roughly **90% of the code generated by AI**.

## Key Takeaways

### 1. Prompt Precision Determines Code Quality

Vague prompts → vague code. Here's the effective prompt pattern I discovered:

```
❌ "Build me a login page"
✅ "Create a login page with email/password fields using shadcn/ui,
   form validation with zod, POST to /api/auth/login, display errors
   at the top of the form, disable button during loading"
```

### 2. Context Is a Scarce Resource

AI context windows are limited. I learned to provide **only relevant file snippets** when needed, rather than dumping the entire project.

### 3. You Still Need to Know How to Code

This was my biggest realization. Vibe Coding isn't "build products without coding skills" — it's "build products faster when you already know how to code." You need to:

- Judge AI-generated code quality
- Spot logical errors
- Understand architectural decisions
- Debug runtime issues

## Conclusion

Vibe Coding is an amplifier, not a replacement. It lets me build 10x faster — but I still need programming knowledge to guide the process.
